PC-PT 0,1,2  3개 static을 DHCP Automatic으로 변경

#라우터에서 동적 IP할당을 처리
Router>en
Router#conf t
Router(config)#ip dhcp pool cho <- cho 부분은 이름이라 아무거나 작성해도(즉, cho가 아니여도 된다고)
#네트워크 주소정보, 서브넷마스크 정보 입력
Router(dhcp-config)#network 192.168.0.0 255.255.255.0
#게이트웨이 주소 입력
Router(dhcp-config)#default-route 192.168.0.1
#dns정보(임의)
Router(dhcp-config)#dns-server 210.111.111.111
Router(dhcp-config)#exit
#컴퓨터에 할당되는 주소중 제외 해야하는 주소 설정
Router(config)#ip dhcp excluded-address 192.168.0.1

설정이 완료된 후 다시 PC-PT 3개 각각을 static으로 바꿨다가 다시 DHCP로 바꿔주면 Gateway와 DNS 부분이 부여됩니다.

Router도 1개 드래그해서 가져옴.

그리고 이 Server는 2번째로 만든 Hub-PT Hub1에 연결해줍니다.
Copper Straight-Through로 Ethernet으로 연결해줍니다.

처음으로 연결된 Router0과의 연결은 끊어주고 새로 드래그한 Router7과 연결해줍니다.

##서브넷 구성
network address : 192.168.0.0
gw : 192.168.0.1
subnet mask : 255.255.255.0
broadcast address : 192.168.0.255
속한클래스 : C클래스
host갯수 : 255개  - networkaddress(1) - gw address(1) - broadcast address(1) = 실제 컴퓨터 배당되는 주소 252개 주소 사용

부서 4개 있을 경우 subnet 구성하시오
255/4 = 64

0-63 :      00000000-00111111 -> 00 | 000000
64-127 :   01000000-01111111 -> 01 | 000000
128-191 :  10000000-10111111 -> 10 | 000000
192-255 :  11000000-11111111 -> 11 | 000000

서브넷마스크 만들시에서 subnet주소까지 비트처리
00000000-00111111 -> 00
255.255.255.192/26

[ 7 or 8개 부서 2^3=8 -> 3bit 서브네트워크 구분되는 비트 ]
4개 부서에 서브넷 구성

subnet network(1) -> 
nwa(192.168.1.0) : submask(255.255.255.192) : gateway(192.168.1.1) : broadcast(192.168.1.127)
: host갯수(64-3=61) 192.168.1.2~126

subnet network(2) -> 
nwa(192.168.1.64) : submask(255.255.255.192) : gateway(192.168.1.65) : broadcast(192.168.1.127) : 
192.168.1.66~126

subnet network(3) ->

subnet network(4) ->
